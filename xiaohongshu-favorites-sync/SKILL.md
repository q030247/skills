# 小红书收藏列表同步

同步用户自己的小红书收藏列表。优先保证登录安全、来源可追溯和增量去重；不绕过访问控制，不代替用户输入账号、密码、验证码或扫码凭证。

## 默认位置

- 收藏笔记：`00-收件箱/小红书/`
- 同步索引：`00-收件箱/小红书/小红书同步索引.md`
- 执行报告：`07-AI产出/整理报告/`

写入知识库前，读取根目录 `AGENTS.md`、`AI-RUNBOOK.md`、`00-收件箱/收件箱说明.md` 和 `知识库规则/SCHEMA.md`。用户指定其他输出位置时仍保留来源信息和隐私边界。

## 1. 探测浏览器能力

检查当前可用工具，不根据工具名称猜测：

1. 优先使用能复用 Cookie/Session、读取页面快照或 DOM、点击和导航的内置浏览器。
2. 优先复用当前小红书标签页或现有浏览器会话，避免新会话丢失登录态。
3. 只有截图能力时可以辅助判断登录界面，但不要声称完成了可靠的列表同步。
4. 没有内置浏览器或等价网页自动化能力时停止：不安装插件、不调用外部抓取服务、不索取 Cookie。输出 Markdown 状态表，提示在具备内置浏览器的环境中重试。

若浏览器工具有强制配套技能或说明，先读取并遵循。

## 2. 判断登录状态

在同一浏览器会话打开 `https://www.xiaohongshu.com`，等待页面稳定，同时检查 URL、可见文本和 DOM，不依赖单一选择器。

“已登录”正向信号：

- 导航栏有“我”或个人主页入口；
- 显示用户昵称或头像；
- “发布”“通知”等个人功能可见；
- 能进入 `/user/profile/{userId}` 并看到“笔记 / 收藏 / 点赞”。

“未登录”信号：

- 出现手机号输入框、扫码登录或“登录探索更多内容”；
- 个人页或收藏页被重定向到登录界面。

信号冲突时记为“待确认”并停止。未登录时停在登录页，请用户手动完成登录；不读取、记录或代填凭证。用户确认后在同一会话复查。登录仍失败或会话过期时停止，不反复重试。

## 3. 进入并加载收藏列表

登录后点击“我”进入个人主页，从 URL 记录 `userId`，切换到“收藏”标签和“笔记”子分类。优先用可访问性定位或文本点击；普通点击失效时，才在页面上下文寻找文本为“收藏”的标签元素并点击。不要用脚本绕过验证码、安全校验或权限。

目标 URL 通常为：`/user/profile/{userId}?tab=fav&subTab=note`。

1. 记录页面显示的收藏总数（若可见）。
2. 逐步滚动并等待懒加载；连续两轮无新增后停止。
3. 从每张卡片提取实际存在的标题、作者、原始链接、笔记 ID、收藏分组/来源分类。
4. 完整保留链接中的 `xsec_token` 和 `xsec_source`。详情访问必须沿用收藏卡片链接；裸 `/explore/{noteId}` 可能被安全校验拦截。
5. 不根据封面猜测缺失字段，缺失值写“待确认”。
6. 用户只要求列表时不逐条打开详情；明确要求正文或列表字段不足时才访问详情。

需要详情时在同一会话打开完整链接，可从 `#noteContainer` 提取可见正文，并检查 `video` 判断媒体类型。没有文字正文时记录“无可提取文字/待人工查看”，不根据图片或视频补造原文。

## 4. 去重与写入

读取同步索引，以笔记 ID 为主键、规范化原始链接为辅助键：

- 不存在：标记“新增”，新建 Markdown 笔记。
- 已存在且无变化：标记“跳过”，不覆盖本地文件。
- 已存在且来源字段有可确认变化：标记“更新候选”；按知识库规则和用户授权决定是否改写笔记。
- 索引标记已删除：标记“跳过（已删除）”，不自动恢复。
- 没有稳定笔记 ID：用原始链接去重并标记“待确认”。

新增笔记至少使用：

```yaml
---
title: 实际标题或待确认标题
summary: 小红书收藏原始输入，内容用途待确认。
tags: [小红书, 收藏]
type: inbox
status: raw
source: xiaohongshu
source_url: 完整原始链接
source_id: 笔记ID
source_collection: 收藏分组或待确认
author: 实际作者或待确认
created: YYYY-MM-DD
updated: YYYY-MM-DD
confidentiality: personal
ai_generated: false
---
```

正文保留实际获取的原始标题、作者、正文/简介和采集时间。不要移动、删除或覆盖已有原始文件。同步索引只修改 `<!-- AI:START -->` 与 `<!-- AI:END -->` 之间的区域。

超过 50 篇时先输出预览和风险，每批最多 50 篇。遇到登录失效、验证码、访问限制、字段冲突或敏感信息时停止当前批次并标记待确认。

### 使用内置脚本

浏览器只负责取得页面中的原始记录，并将对象数组保存为 JSON。字段归一化、去重、索引合并和 Markdown 生成使用技能自带脚本，不要每次临时重写处理代码。三个脚本只依赖 Python 标准库：

| 脚本 | 作用 | 默认安全行为 |
|---|---|---|
| `scripts/normalize_records.py` | 统一字段名、提取笔记 ID、保留安全参数、生成稳定去重键 | 只输出 JSON |
| `scripts/merge_index.py` | 对照索引判断新增、更新候选、跳过和已删除 | 只预览；显式 `--write` 才原子写回 |
| `scripts/render_markdown.py` | 生成 Obsidian 笔记和 Markdown 明细/汇总表 | 只打印报告；显式 `--write` 才新增文件，绝不覆盖 |

从技能目录执行以下流程，路径按当前知识库调整：

```bash
python3 scripts/normalize_records.py browser-records.json -o normalized.json

python3 scripts/merge_index.py \
  "00-收件箱/小红书/小红书同步索引.md" normalized.json \
  --output index-preview.md --records-output batch-with-status.json

python3 scripts/render_markdown.py batch-with-status.json
```

先检查 `index-preview.md`、终端统计和 Markdown 报告。确认记录数不超过 50、没有异常冲突后，先创建新增笔记，再写回索引：

```bash
python3 scripts/render_markdown.py batch-with-status.json \
  --output-dir "00-收件箱/小红书" \
  --report "07-AI产出/整理报告/小红书收藏同步-YYYY-MM-DD.md" \
  --write

python3 scripts/merge_index.py \
  "00-收件箱/小红书/小红书同步索引.md" normalized.json --write
```

创建笔记必须先于索引写回，避免笔记创建失败但索引已登记。若任何写入步骤失败，停止并在报告中记录失败项，不继续伪装成功。

## 5. Markdown 表格输出

最终回复和执行报告都用 Markdown 表格呈现核心结果。即使失败，也必须输出状态表。

成功或部分成功：

```markdown
| 序号 | 标题 | 笔记ID | 作者 | 收藏分组 | 原始链接 | 同步状态 | 本地文件 | 备注 |
|---:|---|---|---|---|---|---|---|---|
| 1 | 示例标题 | abc123 | 示例作者 | 默认收藏 | [打开原文](完整链接) | 新增 | [[文件名]] |  |

| 浏览器能力 | 登录状态 | 发现 | 新增 | 更新候选 | 跳过 | 失败 | 待确认 |
|---|---|---:|---:|---:|---:|---:|---:|
| 可用 | 已登录 | 10 | 6 | 1 | 3 | 0 | 1 |
```

无法执行：

```markdown
| 浏览器能力 | 登录状态 | 同步状态 | 原因 | 用户下一步 |
|---|---|---|---|---|
| 不可用 | 未检查 | 未执行 | 当前智能体没有可复用会话的内置浏览器 | 在具备内置浏览器的环境中重新运行 |
```

## 完成检查

- 使用用户自己的已授权会话，登录由用户手动完成。
- 原始链接保留安全参数，结论可追溯。
- 列表加载到稳定状态，或明确说明只取得部分列表。
- 按笔记 ID 和链接完成索引去重。
- 原始文件未被覆盖、移动或删除。
- 明细和汇总均为 Markdown 表格。
- 报告列出新增、更新候选、跳过、失败和待确认数量。
