# 飞书妙记来源包审计

将本技能用于同步后校验、周日收件箱健康检查和第一阶段连续运行验收。同步由 `feishu-minutes-knowledge-sync` 负责，内容提取由每日或周度处理技能负责。

## 执行前读取

读取知识库根目录的 `AGENTS.md`、`AI-RUNBOOK.md`、`知识库规则/SCHEMA.md`、收件箱说明和飞书同步索引。只读取索引、属性、链接和必要的双链行；不要读取逐字稿正文。

## 审计范围

按来源包而不是文件统计：

1. `minute_token`、`source_id` 或 `source_group_id` 是否重复或冲突。
2. 每个来源包是否恰好包含智能纪要和原始逐字稿。
3. 两篇笔记的 `source_group_id` 是否一致，双向链接是否互指。
4. 索引中的两个路径是否为知识库相对路径且真实存在。
5. 文件整体迁移后，索引中的两个路径是否同步更新。
6. 是否存在单边成功、错误地登记为 `complete`、或失败伪装成成功。
7. 新同步逐字稿是否默认 `transcript_review_status: pending_review`。
8. `ready_for_extraction` 是否设置有效的 `transcript_text_source`。
9. 选择 `corrected` 时，`corrected_transcript` 是否存在、可解析且来源包一致。
10. 未通过人工闸门的来源包是否错误地产生处理记录、候选待办或知识提炼。
11. 是否存在超过 7 天仍无处理记录的来源包。
12. 同步失败、登录失效、格式变化和收件箱持续增长情况。

## 使用审计脚本

优先运行只读脚本：

```bash
python3 <skill-dir>/scripts/audit_source_packages.py \
  --vault <知识库根目录> \
  --index <飞书同步索引相对路径> \
  --json-output <临时JSON路径>
```

脚本只解析索引中的 Markdown 表格和指向的 Markdown 文件，不修改任何文件。若索引结构无法识别，停止并在报告中标记“索引格式待确认”，不要临时猜测或重写索引。

脚本输出后，再根据知识库规则补充同步失败、登录失效、超过 7 天未处理和错误衍生内容等语义检查。

## 严重程度

- `error`：来源包缺文件、路径越界、配对标识冲突、错误登记完整、校订稿来源包不一致。
- `warning`：待人工检查超过阈值、缺少可选字段、链接形式异常、来源更新待确认。
- `info`：正常等待人工检查、已登记墓碑、无新增来源包。

## 写入边界

- 只生成报告，不自动修复索引或文件。
- 不移动、删除、恢复、改名或归档来源包。
- 不把 `deleted` 墓碑恢复为新来源。
- 不读取逐字稿正文，不从审计结果产生任务或知识。
- 中国电信和营销业务的统计与异常分别呈现。

## 输出

默认输出：

`07-AI产出/整理报告/YYYY-Www-飞书妙记来源包审计.md`

报告包含：

```markdown
# 飞书妙记来源包审计

## 摘要
- 来源包总数：
- 完整来源包：
- 错误：
- 警告：
- 等待人工检查：
- 可提取：
- 使用原始文本：
- 使用校订稿：

## 异常明细
| 严重程度 | 来源包 | 问题 | 证据 | 建议动作 |

## 待确认
```

报告设置 `ai_generated: true`。建议动作只是草稿，涉及批量移动、重命名、删除或历史修复时必须另行取得确认。

## 完成检查

- 报告数量和索引、实际文件一致。
- 全程没有读取逐字稿正文或修改知识库源文件。
- 每个异常带来源包标识、文件路径或字段证据。
- 没有把“等待人工检查”误报成“已处理”。
