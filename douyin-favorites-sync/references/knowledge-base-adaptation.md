# 知识库适配规则

本参考把一套经过实践验证的个人知识库工作方式抽象为可发现规则。它描述结构和约束，不包含创建者的用户名、设备名、绝对路径、vault 名称或组织信息。

## 渐进发现

按以下顺序读取，找到足够证据后停止扩大范围：

1. 根目录的 Agent 指令或项目说明。
2. 指令引用的运行手册、schema、术语表或自动化规范。
3. 外部输入/收件箱目录的说明文件。
4. 抖音目标目录中既有的同步索引、模板和 README。
5. AI 衍生内容目录的说明文件。

不要为了找规则扫描整个知识库，也不要默认读取历史归档区。

## 抽象映射

| 需要发现的概念 | 识别证据 | 找不到时的回退 |
|---|---|---|
| 外部输入目录 | 规则声明“外部来源先进入收件箱/Inbox” | 在知识库下使用 `douyin-favorites/` |
| 抖音来源目录 | 路由表或收件箱说明明确映射 Douyin/抖音 | 在外部输入目录下新建语义明确的抖音子目录 |
| 同步索引 | 规则指定路径，或文件正文声明以作品 ID 去重 | 在抖音来源目录创建 `douyin-sync-index.md` |
| AI 报告目录 | 规则声明 AI 生成报告/草稿的专用目录 | 使用抖音来源目录下的 `reports/` |
| AI 写入边界 | 规则定义 AI 标记区或机器维护区 | 不改既有人工文件，另建同步文件 |
| 批次上限 | 批量操作规则中的明确数量 | 每批最多 50 条 |
| 归档排除 | 规则声明历史/归档默认不读 | 默认不读取名称含 archive/归档 的目录 |
| 保密级别 | schema 的枚举和目标目录说明 | 标记“待确认”，不猜测业务归属 |

路径命名应服从目标知识库，不要把上表的英文回退名误当成强制目录名。

## 来源笔记约束

抖音收藏属于外部源数据。同步列表至少应保留：

- 来源类型
- 作品链接与稳定作品 ID
- 作者
- 收藏合集
- 来源发布时间和本次采集时间
- 标题或简介原文
- 同步状态

目标 schema 若定义 `source_url`、`source_id`、`source_collection` 等同义字段，优先使用目标字段名。不存在时再使用技能模板，不要修改目标 schema。

标题或简介只能来自页面/API 的原始字段，不能根据封面推断。知识库如果要求记录“为什么收藏”或“支持什么决策/输出”，保留对应列并写“待确认”，等待人工整理。

## 人工内容与 AI 内容

- 原始记录和人工补充不可覆盖。
- AI 只更新目标规则明确标记的机器维护区域。
- 没有标记区时另建同步列表或报告。
- 分类、移动、建立实体页和业务结论属于后续整理，不属于同步动作。
- 同步失败、字段缺失和登录问题写入报告，不伪装成成功。

## 索引状态

索引永远保留历史作品 ID，并至少区分：

| 状态 | 含义 | 默认动作 |
|---|---|---|
| `added` | 本轮首次同步 | 写入表格与索引 |
| `updated` | 来源字段确认发生变化 | 只更新 AI 可维护区 |
| `skipped` | 已存在且未变化 | 不重复写入 |
| `local_missing` | 本地记录被人工删除或缺失 | 保留 ID，不自动恢复 |
| `source_missing` | 完整遍历后来源不再出现 | 保留本地记录，不物理删除 |
| `failed` | 采集或落盘失败 | 不登记为成功，写入报告 |

只有用户明确要求恢复时，才重建 `local_missing` 条目。

## 完成报告

报告是 AI 衍生内容，采用 Markdown 表格记录：执行环境、登录结果、同步范围、原始条数、去重条数、新增、更新、跳过、失败、待确认、输出路径和安全检查。数量必须能与索引和收藏表格相互核对。
