# 浏览器路径

## 自动路径所需能力

必须同时具备：持久化会话、DOM 检查、页面脚本执行或网络响应读取。仅能打开网址、截图或搜索公开网页不足以读取私人收藏。

自动路径的核心不是调用抖音接口，而是让响应观察器在收藏页首屏请求之前生效，再通过正常滚动让页面自身加载数据。观察器不得读取请求头、Cookie 或修改业务响应。

优先使用浏览器级初始化脚本或网络响应监听，它们能在新文档业务脚本之前运行。若目标收藏页已经加载，先注册初始化脚本再刷新；普通页面 `evaluate` 注入的包装器会在刷新时丢失，不能把“已注入后刷新”当作成功，除非工具明确提供跨导航注入能力。

没有跨导航初始化能力时，可以在同一 SPA 文档的上一级页面安装观察器，再通过站内操作进入收藏页，并在进入后检查包装器和首个 `listcollection` 响应是否仍存在。若发生整页导航导致观察器失效，自动路径不具备可靠的首屏捕获能力，应改用页面首屏 JSON 兜底或人工导出。

## 未登录处理

打开抖音登录界面，提示用户亲自扫码。不要截图、OCR、转发或保存二维码。用户完成后重新打开收藏页，以头像/昵称、收藏入口和无登录重定向共同验证。

## 人工导出路径

如果没有完整浏览器能力，向用户说明：

1. 在自己的浏览器登录抖音，但先不要进入目标收藏合集；若已经进入，准备在安装观察器后刷新。
2. 打开浏览器开发者工具，在用户明确理解并同意的前提下运行本技能提供的本地导出脚本。
3. 脚本只应捕获 `listcollection` 的响应体、滚动页面、合并 `aweme_list` 并下载 JSON。安装完成后再进入收藏合集；如果脚本不能跨刷新持续存在，则通过同一页面的站内导航进入，不要先刷新再误以为仍在拦截。
4. 不要求用户复制 Cookie、sessionid、ttwid、请求头或个人凭据。
5. 用户把 JSON 放入授权目录后，智能体校验顶层数组或 `aweme_list` 结构、作品 ID 和条数，再继续转换。

若平台禁止提供或运行页面脚本，就指导用户用开发者工具 Network 面板筛选 `listcollection`，将响应另存为 JSON。不要引导绕过验证码或反爬机制。

## 完整性检查

导出或自动采集都要记录：页面显示总数、当前运行唯一 ID 数、索引原有 ID 数、合并唯一 ID 数以及差额。只有合并数与页面总数一致且分页结束时，才可以报告完整同步。数量不一致时保留已成功数据并标记部分完成；索引去重只能避免重复写入，不能补回从未捕获的数据。
