# 浮墨每日整理

## 目标

只处理已经进入知识库的浮墨笔记，在不改写源数据的前提下形成可复核的分流、行动候选、闪念卡、文章卡和每日处理报告。

本 skill 不执行外部同步，也不删除、移动、归档或覆盖来源笔记。

## 开始前读取

依次读取：

1. `AGENTS.md`
2. `AI-RUNBOOK.md`
3. `知识库规则/SCHEMA.md`
4. `01-驾驶舱/候选待办.md`
5. `07-AI产出/知识提炼/浮墨闪念/README.md`
6. `07-AI产出/知识提炼/文章提炼/README.md`
7. `00-收件箱/浮墨笔记/浮墨同步索引.md`

只在需要判断归属或去重时读取相关项目主页、领域主页、`知识库规则/GLOSSARY.md`和现有资源标题。不要扫描`99-归档/`。

## 输入范围

从浮墨同步索引中选择满足任一条件的来源笔记：

- 本次时间窗口内新增；
- 来源发生更新；
- `ai_processing_status: unprocessed`；
- `article_extraction_status: pending`。

先读取索引、属性、标题和摘要，再按处理需要读取正文。已经稳定处理且来源未变化的笔记跳过。

## 工作流

### 1. 检查已批准候选

在开始新的抽取前，检查来源笔记中已经勾选的`#候选待办`。

仅在以下条件全部满足时转换为正式任务：

- 候选已勾选；
- 有稳定`candidate_id`；
- 没有`#已转正式任务`；
- 没有`promoted_to`；
- 建议正式位置唯一且有效；
- 与现有正式任务不重复。

转换成功后：

1. 在唯一所属项目或领域中创建一条`#task`，包含领域标签、来源双链和验收标准。
2. 在来源候选行添加`#已转正式任务`。
3. 记录`promoted_to`和`promoted_at`。

勾选只表示批准转换，不表示原工作已经完成。目标不明确、链接失效或疑似重复时不要转换，写入报告的“待确认”。

### 2. 多标签分流

一条来源可以同时拥有多个类型：

- `flash`：闪念、问题、机会、产品设想、内容选题或待验证假设；
- `candidate_task`：包含可执行动作，但尚未获得用户确认；
- `article_link`：包含文章、图文、视频页面或其他外部链接；
- `other`：暂时无法归类或只需保留。

只更新来源笔记的`<!-- AI:START -->`至`<!-- AI:END -->`区域，以及以下AI管理属性：

```yaml
capture_types: [flash, candidate_task, article_link]
ai_processing_status: triaged
article_extraction_status: pending
```

保留原始内容、来源时间、来源标签和原始链接。来源同步更新不能覆盖AI区域和AI管理属性。

### 3. 生成闪念卡

对`flash`提炼：核心想法、要解决的问题、潜在使用者、潜在价值、最小验证方式、不确定之处及关联项目或领域。

- 输出到`07-AI产出/知识提炼/浮墨闪念/`。
- 卡片遵循该目录README和知识库属性规范。
- 必须链接原始浮墨，并以`source_id`防止重复生成。
- 成熟度只使用`待观察`、`待验证`、`可沉淀`。
- Vibe Coding想法跟随其服务项目。
- 未经人工确认，不写入`04-资源/`，不创建正式项目。

### 4. 生成候选待办

只抽取具体、可执行的动作。负责人和截止日期仅采用原文明确信息；相对日期同时保留原始表达，并可依据`source_created_at`给出建议日期。

在来源笔记AI区域使用：

```markdown
- [ ] #候选待办 #领域标签 具体行动 <!-- candidate_id: flomo-来源ID-序号 -->
  - 建议正式位置：[[项目或领域文件]]
  - 验收标准：
  - 原始时间表达：
  - 候选状态：待确认
```

候选不得添加`#task`。`candidate_id`应由稳定来源ID和稳定序号组成；重复运行时复用原候选，不重复创建。

### 5. 提炼链接和概念

1. 提取并标准化URL，去除不影响正文的追踪参数，以标准化URL去重。
2. 只读取公开可访问或用户已经授权访问的页面。
3. 每次最多处理10个新链接，超出部分顺延。
4. 记录正文状态：`accessible`、`partial`或`blocked`。
5. 无法取得正文时，不根据标题、缩略图或推荐文案猜测；标记`blocked`并记录原因。
6. `blocked`链接只有在来源更新、访问条件变化或用户重置为`pending`时才重试。
7. 正文可读取时，在`07-AI产出/知识提炼/文章提炼/`生成文章卡，包含摘要、论点、依据、适用场景、限制和不超过5个概念候选。
8. 抽取概念前搜索`04-资源/`和`知识库规则/GLOSSARY.md`。已有概念生成增量补充建议；新概念留在文章提炼目录等待确认。
9. 不完整复制受版权保护的内容，只保留来源链接和支持结论所需的最小证据。

## 输出报告

生成：

`07-AI产出/每日整理报告/YYYY-MM-DD-浮墨分流报告.md`

报告遵守`SCHEMA.md`，并至少包含：

- 处理时间窗口；
- 新增分流数；
- 闪念数；
- 候选待办数；
- 已批准并成功转换数；
- 重复跳过数；
- 文章链接数；
- 提炼成功数；
- 正文受阻数；
- 概念候选数；
- 待确认项；
- 失败及原因。

## 完成检查

- 原始浮墨只有一份，AI区域外没有变化。
- 每个衍生文件都能追溯到`source_id`和原始浮墨。
- 没有未经批准的正式`#task`。
- 已转换候选通过三重幂等信息防止再次转换。
- 没有根据不可访问页面猜测正文。
- 没有移动、删除、归档来源笔记，也没有对外发送知识库内容。
