# flomo-daily-processing

对已经进入本地知识库的 flomo（浮墨）笔记执行每日整理，不负责从外部服务同步数据。

## 适用场景

- 整理新增或待处理的浮墨笔记；
- 将一条笔记分流为闪念、候选待办、文章链接或其他类型；
- 生成可追溯的闪念卡和文章提炼卡；
- 把用户已经批准、目标位置唯一的候选待办转换为正式任务；
- 生成每日浮墨分流报告。

## 核心边界

- 不执行浮墨同步，建议与 `flomo-knowledge-sync` 配合使用；
- 不改写原始内容、来源时间、来源标签和原始链接；
- 未经人工批准，候选行动不得成为正式任务；
- 正文不可访问时不根据标题或缩略图猜测内容；
- 不删除、移动、归档来源笔记，也不把知识库内容发送到外部服务。

## 是否包含脚本

本技能当前不包含专用脚本。多标签分类、知识关联、闪念成熟度判断和文章概念提炼依赖 Agent 的语义理解；稳定 ID、幂等信息、批量上限和报告字段由 `SKILL.md` 约束。

## 典型触发方式

- “整理今天新增和待处理的浮墨笔记，不要执行同步。”
- “提取浮墨里的闪念、候选待办和文章链接。”
- “检查我已经勾选的浮墨候选，把符合条件的转换为正式任务。”
