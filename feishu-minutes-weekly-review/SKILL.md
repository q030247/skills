# 飞书妙记周度复盘

将本技能用于每日处理完成后的聚合与知识增量建议。它不负责同步、不负责人工校订、不替代来源包审计。

## 执行前读取

读取知识库根目录的 `AGENTS.md`、`AI-RUNBOOK.md`、`知识库规则/SCHEMA.md`、`知识库规则/GLOSSARY.md`、AI产出说明，以及本周飞书每日整理报告。

只处理本周已经满足以下条件的来源包：

- `transcript_review_status: ready_for_extraction`
- `transcript_text_source` 为 `original` 或有效的 `corrected`
- 已存在每日整理记录
- 来源包配对完整且没有未解决的严重审计异常

未通过条件的来源包只计入“待处理”，不能参与主题、决定、候选行动、培训或知识提炼。

## 输入顺序

1. 本周每日飞书整理报告及其属性、摘要。
2. 报告链接的智能纪要和已确认结果。
3. 现有资源标题、术语表、相关项目和领域主页。
4. 只有具体结论必须核验且智能纪要不足时，才读取该来源包选定的逐字稿文本。

不要扫描本周或历史全部逐字稿，不读取 `99-归档/`。

## 业务隔离

分别处理并呈现：

- 中国电信：企业征信、数据口径、问题模式、交付方法、联合建模。
- 营销业务：客户痛点、渠道效果、电销话术、口腔/中医/养生/医美规律、医院和数交所合作。
- 斌珊工作室：观点、案例、选题和可转化素材。
- 其他个人领域：股票量化、健康等。

不得混合中国电信与营销源数据。营销主体只能使用来源明确出现的域舟智能、维够智能、美荔管家、赛勒王或其他明确主体；不确定时写“待确认”。Vibe Coding 归入它服务的项目。

## 聚合方法

1. 按项目、领域和问题聚类，保留每条来源双链。
2. 标出本周重复出现两次以上的主题，但不因重复自动判定正确。
3. 区分事实、明确决定、候选行动、假设、经验和待确认。
4. 搜索 `04-资源/` 与术语表：已有条目提出增量补充，没有条目时才建议新建。
5. 对培训、话术、案例和问答，只保留支持结论所需的最小证据，不复制整篇逐字稿。
6. 审计候选待办是否仍为候选、是否已获用户批准、是否已由下一次 `feishu-minutes-knowledge-sync` 转换；不在本技能中转换正式任务。

默认输出：

`07-AI产出/知识提炼/YYYY-Www-飞书妙记知识提炼建议.md`

使用结构：

```markdown

- 已整理来源包：
- 待人工检查：
- 校订中：
- 审计异常：
- 使用原始文本：
- 使用校订稿：

## 中国电信
### 重复主题与变化
### 已有知识的增量建议
### 新知识候选

### 新知识候选

## 斌珊工作室及其他领域

```

报告设置 `ai_generated: true`。每个知识候选说明来源、适用场景、限制、建议目标条目和需要人工确认的内容。

## 写入边界

- 不直接修改 `04-资源/`、项目状态、负责人或截止日期。
- 不创建正式 `#task`，不根据会议中的“完成了”自动勾选任务。
- 不移动、删除、覆盖或归档来源包。
- 不对外发送知识库内容。

## 完成检查

- 所有参与提炼的来源包均通过人工校订闸门和每日处理。
- 中国电信与营销业务保持数据隔离。
- 已有知识优先获得增量建议，没有大量空实体页。
- 每条结论和建议都有来源双链，逐字稿没有被整篇复制。
- 不确定的主体、负责人、日期、金额和业务结论均进入“待确认”。
