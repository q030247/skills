# 飞书妙记每日处理

将本技能用于“同步之后、人工确认之前”的整理环节。同步由 `feishu-minutes-knowledge-sync` 负责；本技能不访问飞书、不重新下载妙记。

## 执行前读取

从知识库根目录依次读取：

1. `AGENTS.md`
2. `AI-RUNBOOK.md`
3. `知识库规则/SCHEMA.md`
4. `知识库规则/GLOSSARY.md`
5. 收件箱说明、飞书同步索引和相关项目或领域主页

只处理本次时间窗口内新增、更新或尚无处理记录的来源包。默认批次不超过知识库规定的上限；没有规定时最多 50 个来源包。

## 来源包识别

1. 优先使用 `source_group_id`，兼容旧数据时使用 `minute_token` 或 `source_id`。
2. 将智能纪要和原始逐字稿视为一个来源包，不能分别计数或分别得出结论。
3. 任一文件缺失、两个文件的来源标识不一致、索引路径失效时停止该来源包，将原因写入“待确认”。
4. 读取顺序为：索引 → 两篇笔记的属性和摘要 → 人工校订闸门 → 智能纪要正文 → 必要时选定的逐字稿正文。

## 人工校订闸门

读取逐字稿属性，但在闸门通过前不要读取逐字稿正文，也不要抽取任何结论。

| 状态 | 行为 |
|---|---|
| `pending_review` | 记录“等待人工检查”，停止处理来源包 |
| 其他非 `ready_for_extraction` 状态 | 记录当前状态，停止处理来源包 |
| `ready_for_extraction` + `transcript_text_source: original` | 使用原始逐字稿作为核验文本 |
| `ready_for_extraction` + `transcript_text_source: corrected` | 校验并使用 `corrected_transcript` 指向的人工校订稿 |

选择校订稿时，确认文件存在、双链有效且 `source_group_id` 与来源包一致。字段缺失或校订稿异常时停止该来源包。原始逐字稿正文不得校改。

## 多标签分类

一个来源包可以同时属于多个类型，分类不得互斥，也不得为了对齐其他来源而删减飞书原有提取能力。使用以下稳定类型：

- `conclusion`：核心结论。
- `decision`：明确决定。
- `candidate_task`：候选待办。
- `business_lead`：业务线索。
- `training`：培训材料、步骤、案例或问答。
- `knowledge`：可复用知识、方法或规律。
- `content`：内容素材、选题、观点或案例建议。
- `flash`：新增类型；闪念、机会、问题或待验证假设。
- `article_link`：新增类型；值得单独提炼的外部文章或链接。
- `other`：新增类型；有价值但暂不适合以上类型的内容。

在智能纪要的 AI 受控区或衍生处理卡记录 `capture_types`。没有命中的类型不强行填写；同一段证据可支撑多个类型，但每个结论都必须保留来源包链接。

更新`ai_processing_status`、`article_extraction_status`或其他状态字段时，同时维护对应`*_label`，格式为`中文 / English`；判断与幂等逻辑只读取英文机器值。

## 提取流程

闸门通过后默认先读智能纪要。只有为核实决定、负责人、日期、金额、事实，或提炼培训步骤、案例、问答时，才读取选定的逐字稿文本。

提取以下内容：

- 核心结论：区分原文事实、AI归纳和待确认推断。
- 明确决定：只有来源明确表达为决定时才记录。
- 闪念、机会和待验证假设：标明证据与待验证内容，不冒充会议决定。
- 候选待办：只生成候选，不添加 `#task`。
- 业务线索：不猜测客户意向或成交结论。
- 建议归属和候选项目：Vibe Coding 跟随其服务项目。
- 可沉淀知识、培训材料、案例、问答和内容素材建议；这些原有分类必须保留。
- 外部文章或链接：保留原链接并提出文章卡建议；只有公开可访问或用户已授权读取时才提炼正文，无法访问时标记待确认，不猜测内容。
- 其他有价值内容：说明为什么保留，以及建议的后续归属。
- 待确认：负责人、日期、金额、主体、业务归属等不确定信息。

中国电信和营销业务必须保持源数据隔离。营销主体只能采用来源明确出现的名称。

## 候选待办格式

使用稳定 ID，重复运行不得重复生成：

```markdown
- [ ] #候选待办 #领域标签 具体行动 <!-- candidate_id: feishu-<minute_token>-<序号> -->
  - 建议正式位置：[[项目或领域文件]]
  - 验收标准：
  - 原文明确负责人：
  - 原始时间表达：
  - 候选状态：待确认
```

飞书妙记自带的 todos 仍是来源内容，不能直接视为知识库正式任务。合同、财务、合规、客户触达、经营决策和生产系统行动必须由用户确认。

用户在候选驾驶舱勾选候选表示批准转为正式任务，不表示行动已经完成。本技能保留勾选状态并报告“已批准、等待下一次飞书同步转换”；转换由 `feishu-minutes-knowledge-sync` 执行，包括没有新增妙记的同步。

## 写入边界

- 不移动、删除、覆盖智能纪要或逐字稿。
- 只有智能纪要存在 `<!-- AI:START -->` 与 `<!-- AI:END -->` 时，才可幂等更新该区域；不得向原始逐字稿写入提炼结果。
- 旧智能纪要没有受控标记时，把衍生结果写入知识库规定的 AI 产出目录；没有规定时使用 `07-AI产出/飞书妙记处理/`。衍生卡必须带来源包双链、稳定 `candidate_id` 和相同的多标签分类，使候选驾驶舱可以统一查询来源笔记与衍生卡。
- 未经确认，不迁出收件箱、不修改项目状态、不自动创建正式任务。
- 报告设置 `ai_generated: true`，并保留来源包双链。

## 每日报告

默认输出到知识库规定的每日整理报告目录；当前知识库建议使用：

`07-AI产出/每日整理报告/YYYY-MM-DD-飞书妙记整理.md`

每个来源包使用：

```markdown
## 来源标题

- 来源包或来源ID：
- 使用文本：智能纪要 / 原始逐字稿 / 人工校订稿
- 建议归属：
- 候选项目：
- 内容分类：
- 核心结论：
- 已确认决定：
- 闪念、机会或假设：
- 候选待办：
- 业务线索：
- 培训材料：
- 可沉淀知识：
- 可转化内容：
- 文章或链接：
- 其他：
- 待确认：
```

汇总待人工检查、正在校订、可提取、使用原始文本、使用校订稿、校订稿异常、已整理和失败数量。未通过闸门的来源包不计入“已整理”。

## 完成检查

- 统计单位是来源包，不是文件。
- 未放行逐字稿没有产生任何闪念、决定、候选待办、培训、线索或知识。
- 每条衍生结论可以追溯到来源包。
- 没有覆盖源数据，也没有把候选行动冒充正式任务。
- 飞书原有的结论、决定、候选待办、业务线索、培训、知识和内容素材分类均未减少；新增类型可以与它们并存。
- 已勾选候选保留批准状态，并被列为等待飞书同步转换。
- 重复运行不会重复生成处理卡或候选待办。
